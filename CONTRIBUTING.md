# Contributing Guidelines

Thank you for your interest in contributing to **Liquid C2** projects.

We welcome constructive, responsible contributions that align with our engineering and security standards.

---

## 🧭 General Guidelines

Before contributing, please ensure that:

- You have read the repository documentation
- Your contribution is relevant and well‑scoped
- No sensitive data is included (credentials, tokens, customer data)
- Your work complies with applicable laws and ethical guidelines

---

## 🛠️ How to Contribute

Depending on the repository, you may contribute by:

- Reporting bugs
- Suggesting improvements
- Submitting pull requests
- Improving documentation

Always check for repository‑specific rules that may override this document.

---

## 🔀 Pull Requests

When submitting a pull request:

- Clearly describe **what** you are changing and **why**
- Keep changes focused and minimal
- Follow existing code style and conventions
- Ensure your changes do not weaken security controls
- Reference related issues where applicable

Incomplete or unclear PRs may be closed without review.

---

## 🐞 Reporting Issues

- Use issue templates when available
- Provide clear reproduction steps
- Do **not** report security vulnerabilities publicly  
  → Follow the instructions in `SECURITY.md`

---

## 🔐 Security Considerations

Security is critical to everything we publish.

- Do not introduce known vulnerabilities
- Avoid unsafe defaults
- Assume code may be used in production environments

If you are unsure, ask before submitting.

---

## 🤝 Code of Conduct

All contributors must follow the organization’s Code of Conduct.

Harassment, abuse, or unprofessional behavior will not be tolerated.

---

Thank you for helping improve Liquid C2 projects.
